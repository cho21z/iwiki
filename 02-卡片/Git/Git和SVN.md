Git 和 SVN 核心差异在于架构，SVN 是集中式的，Git 是分布式的。

## Git
* 完整副本：每个用户本地都有一份完整的仓库副本，包含所有历史记录、分支等。
* 离线工作：绝大多数操作，比如查看日志、创建分支、提交代码等，都可以在本地完成，无需联网。
* 工作流程：

        clone 远程仓库 → 本地修改 → add 暂存 → commit 到本地仓库 → push 到远程

## SVN
* 单服务器：所有的版本数据全都存储在一个中央服务器上。
* 网络受限：必须联网才能查看历史记录、创建分支、提交代码，一旦服务器宕机，无法进行协作，且数据可能丢失。
* 工作流程：

        中央服务器 checkout → update 拉取最新 → 本地修改 → commit 到中央服务器

综合看来，Git 除学习成本高外，其余均优于 SVN，且已成为行业最主流的版本控制工具。
