> 💡 GET / POST / PUT / PATCH / DELETE 分别该在什么场景下用？PUT 和 PATCH 有什么区别？

在 RESTful API 中，URL 负责定位资源，具体动作交给 HTTP Method 表达。

## 常用方法
| Method | 含义 | 典型用途 |
| --- | --- | --- |
| GET | 获取资源 | 查询列表、查询详情 |
| POST | 创建资源 | 新增记录、触发创建型操作 |
| PUT | 整体更新资源 | 完整覆盖更新 |
| PATCH | 部分更新资源 | 修改部分字段 |
| DELETE | 删除资源 | 删除指定资源 |

## 使用注意
- **GET 必须是安全的**，不应产生副作用，不要把删除、下单、扣库存这类动作塞进 GET，否则可能被缓存、预加载、爬虫等误触发。
- **PUT vs PATCH**：PUT 是完整替换，请求体要包含资源的完整表示；PATCH 只更新传入的字段，更轻量。如果系统不区分，也可以统一用 PUT，但要在文档中写清语义。
- **DELETE 的语义是“对外被移除”**，底层是物理删除还是逻辑删除都可以，API 层面无需区分。
