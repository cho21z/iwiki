> 💡 一个业务对象要具备什么特征才能当作"资源"？那些"动作型"场景怎么办？

判断一个业务对象是否可以看作资源，最简单的办法是看它是否可以被**唯一标识**、被 **CRUD**。

例如：用户、订单、商品、评论，都是典型资源。

## 动作型场景如何资源化
像 `login`、`refreshToken`、`sendSms`、`payOrder` 这类看起来更像"动作"的场景，可以进一步抽象成资源：

| 动作 | 资源化表达 |
| --- | --- |
| 登录 | 创建一个会话：`POST /sessions` |
| 发送短信 | 创建一条短信发送任务 |
| 支付订单 | 创建一笔支付：`POST /payments` |

RESTful 本质上是把思维从**动作思维**切换到**资源思维**。
