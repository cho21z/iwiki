> 💡 列表接口的分页、过滤、排序、搜索条件应该怎么传？分页接口除了数据还要返回什么？

分页、过滤、排序、搜索是列表类接口最常见的四种查询需求，它们都应该通过查询参数（query string）来表达，而不是写进 URL 路径。

## 常用参数约定
| 参数 | 说明 |
| --- | --- |
| page | 页码 |
| page_size | 每页数量 |
| filter / 具体字段名 | 过滤条件 |
| sort_by | 排序字段 |
| sort_order | 排序方向（asc / desc） |
| keyword | 搜索关键词 |

组合使用示例：
```http
GET /orders?status=paid&page=1&page_size=20&sort_by=created_at&sort_order=desc
```

## 分页接口要返回元信息
除了列表数据外，分页接口通常还要返回分页元信息，便于前端渲染分页组件：
```json
{
  "page": 1,
  "page_size": 10,
  "total": 20,
  "has_more": true,
  "data": [
    {
      "id": 1,
      "name": "user1"
    }
  ]
}
```
