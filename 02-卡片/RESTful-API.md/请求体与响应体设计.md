> 💡 请求体和响应体的字段要怎么命名？错误响应应该长什么样？

请求体表达的是"客户端想提交什么"，响应体表达的是"服务端处理后的结果"。

## 字段命名风格统一
要么使用 snake_case，要么使用 camelCase，**不要混用**。同一个系统里如果出现 `user_id` 和 `userName` 交错，会显著增加沟通和维护成本。

创建订单的请求体示例：
```json
{
  "user_id": 1001,
  "items": [
    {
      "product_id": 2001,
      "quantity": 2
    }
  ],
  "address_id": 3001
}
```

## 错误响应结构要稳定
不要一会儿返回 `msg`，一会儿返回 `error_message`。建议统一错误结构：
```json
{
  "code": "INVALID_PARAMETER",
  "message": "page 必须大于 0",
  "details": {
    "field": "page"
  }
}
```

- HTTP 状态码表示协议层结果
- 业务码（`code`）表示更细粒度的业务语义
- 两者配合使用，而不是用业务码替代状态码
