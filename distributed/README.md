# 微服务与分布式

> 让多个服务/进程协同工作，并在故障、网络分区、流量洪峰下保持可用与正确。

对应学习阶段：**阶段 5** —— 微服务 / 分布式 / 稳定性（8-10 周）

---

## 📍 学习路径

```
阶段一：分布式基础理论（先建立心智模型）
  ├─ CAP / BASE
  ├─ 一致性模型（强一致、最终一致、线性一致）
  ├─ 时钟与因果（Lamport、Vector Clock）
  └─ 故障模型（节点宕机、网络分区、拜占庭）

阶段二：微服务工程基本盘
  ├─ 服务拆分原则
  ├─ 服务注册与发现（etcd、Nacos、Polaris）
  ├─ 配置中心
  ├─ API 网关
  └─ 微服务框架（Kratos、go-zero、Kitex）

阶段三：服务治理（流量进来之后怎么办）
  ├─ 限流（令牌桶、漏桶、滑动窗口）
  ├─ 熔断与降级
  ├─ 超时、重试、幂等
  ├─ 负载均衡
  └─ 灰度发布与回滚

阶段四：分布式核心问题（难点集中区）
  ├─ 分布式锁
  ├─ 分布式事务（2PC、TCC、Saga、本地消息表）
  ├─ 分布式 ID
  ├─ 一致性算法（Raft、Paxos 简介）
  └─ 分布式调度与任务补偿

阶段五：稳定性与高可用
  ├─ SLI / SLO / SLA
  ├─ 容灾与故障恢复
  ├─ 容量规划与压测
  └─ 事故复盘方法论
```
