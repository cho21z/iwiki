# 安全

> 安全不是"上线之后再补"的事——它是后端工程师必须从设计阶段就在脑子里持续运行的那条隐线。这个目录沉淀的是"工程上下功夫的安全"，不是密码学论文。

对应学习阶段：**贯穿阶段 3 之后**，建议在能写出 API 服务后立即开始积累。

---

## 📍 学习路径

```
阶段一：身份认证（你是谁）
  ├─ 密码存储与哈希算法
  ├─ Session vs Token vs JWT
  ├─ OAuth 2.0 / OIDC
  ├─ 单点登录 SSO
  └─ 二次验证 2FA / MFA

阶段二：访问控制（你能做什么）
  ├─ 鉴权模型（RBAC / ABAC / ReBAC）
  ├─ 越权问题（水平 / 垂直）
  ├─ API 网关层鉴权
  └─ 微服务间的信任传递

阶段三：常见 Web 攻击与防御
  ├─ SQL 注入
  ├─ XSS / CSRF
  ├─ SSRF / 命令注入
  ├─ 文件上传攻击
  └─ 序列化攻击

阶段四：传输与存储安全
  ├─ HTTPS / TLS
  ├─ 数据加密（对称 / 非对称 / 哈希）
  ├─ 敏感信息脱敏
  ├─ 密钥管理（KMS / Vault）
  └─ 日志与监控中的敏感数据治理

阶段五：业务安全（被忽视但很重要）
  ├─ 限流与防刷
  ├─ 防重放与签名机制
  ├─ 风控与异常行为识别
  ├─ 接口幂等与防篡改
  └─ 安全审计与合规
```
