# 系统设计

> 把"业务需求"翻译成"可落地、可演进、能扛住流量"的技术方案。这是所有前序知识的综合应用场。

对应学习阶段：**贯穿整个学习过程，阶段 5 之后重点积累**

---

## 📍 学习路径

### 阶段一：方法论与套路（认知打底）
- 系统设计的 4S 分析法（Scenario / Service / Storage / Scale）
- 容量估算（QPS、存储、带宽的量级感）
- 读写比例与访问模式分析
- 一致性、可用性、分区容忍性（CAP）的取舍语言
- 同步 vs 异步、推 vs 拉的决策依据

### 阶段二：核心组件的选型与组合（积木拼装）
- 存储选型（RDB / NoSQL / KV / 列存 / 搜索 / 时序）
- 缓存分层（本地缓存 / 分布式缓存 / CDN）
- 消息队列在系统中的位置（解耦 / 削峰 / 异步 / 广播）
- 数据库分库分表 vs 读写分离 vs 多级缓存
- 同步链路 vs 异步链路的边界划分

### 阶段三：经典场景设计（套路熟练）
- 秒杀 / 抢购系统
- Feed 流 / 时间线（推模式 / 拉模式 / 推拉结合）
- 短链 / 发号器
- 排行榜 / 计数器
- 评论 / 点赞 / 关注关系
- 通知 / 即时消息
- 搜索 / 推荐前置链路

### 阶段四：高阶能力（架构思维）
- 多机房 / 异地多活
- 容灾与故障演练
- 业务建模与领域抽象
- 平台化与中台化思考
- 系统演进路径（从单体到分布式的合理节奏）
