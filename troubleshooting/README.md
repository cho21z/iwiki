# 故障排查与稳定性

> 写代码的工程师很多，能"在凌晨三点把线上事故定位到根因"的工程师不多。这个目录沉淀的是后端工程师的"硬实战层"——从现象到根因的完整链路。

对应学习阶段：**贯穿阶段 5-6**，建议在有真实服务可观测性后开始系统积累。

---

## 📍 学习路径

```
阶段一：方法论（建立排障的"思维框架"）
  ├─ 自顶向下还是自底向上
  ├─ USE / RED / 黄金信号方法
  ├─ 先止血再定位的优先级
  └─ 假设-验证的循环节奏

阶段二：CPU 类问题（最常见的性能问题）
  ├─ pprof 的 CPU profile 怎么读
  ├─ 火焰图与采样原理
  ├─ 锁竞争与上下文切换识别
  └─ Go 调度器异常现象

阶段三：内存类问题（最难定位的一类）
  ├─ heap profile 与 inuse / alloc 区别
  ├─ goroutine 泄漏
  ├─ 大对象与内存碎片
  └─ OOM 的现场保留

阶段四：I/O 与网络类问题（链路最长）
  ├─ 慢 SQL 全链路追踪
  ├─ TCP 连接异常（CLOSE_WAIT / TIME_WAIT）
  ├─ 缓存击穿与雪崩的现场识别
  └─ 跨服务超时的根因定位

阶段五：事故响应与复盘（事故的工程化）
  ├─ 事故分级与升级流程
  ├─ 现场保留与证据链
  ├─ 5 Why 与故障树分析
  └─ 复盘文档与改进项跟踪
```
